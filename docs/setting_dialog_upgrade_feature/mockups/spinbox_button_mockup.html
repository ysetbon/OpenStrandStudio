Settings Dialog — Spinbox +/− Button Upgrade (v1.109)

Three proposals for replacing the native Qt spinbox arrows across the whole settings dialog. Every option is shown in the three app themes and in LTR + RTL (Hebrew). Buttons are live — click or press-and-hold to repeat (same behavior as the Edit Strand Angles dialog).
OPTION A Angle-dialog twins — separate flat − / + buttons
Exact visual language of the existing Edit Strand Angles buttons (flat, 3px radius, same colors/hover/pressed). Buttons flank the value field: [−] [value] [+], mirrored in RTL. Most consistent with what users already know.
OPTION B Segmented stepper — one joined control
A single rounded container [ − | value | + ] with hairline separators. Reads as one control, modern look, smallest visual noise. Matches the 4–5px radius of other settings controls.
OPTION C Compact trailing pair — stacked + / − at the field edge
Keeps the exact footprint of today's QSpinBox (buttons at the trailing edge, mirrored to the left side in RTL), but with clear + on top and − below instead of the tiny native arrows. Least layout change.
SHIPPED Arrow Sizes dropdown — layer context menu
Added to the arrow section of the layer right-click menu: a combo-styled Arrow Sizes → Adjust dropdown exposing all six settings-dialog arrow numbers with the chosen Option B steppers. Click Adjust to open the panel — values step live in the app and persist to the settings file.
Implementation note: all three options can be done purely with Qt stylesheets + a small ThemedSpinBox helper applied dialog-wide (Options A/B wrap the spinbox in a tiny widget; Option C is stylesheet-only on QSpinBox::up-button/down-button). Press-and-hold auto-repeat is native QSpinBox behavior and is kept in every option.
Shipped: Option B was implemented (110px control, halved value gap), plus the Arrow Sizes dropdown above in the layer context menu.

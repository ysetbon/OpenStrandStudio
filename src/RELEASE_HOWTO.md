# Release How-To (v1.110 and onward)

How to produce all the version-specific release files (Windows Inno Setup + macOS
installers + in-app What's New), exactly the way it was done for 1.108 and 1.109.

## What a release touches

| File | What changes |
|---|---|
| `src/translations.py` | The 7 `whats_new_info` blocks (in-app "What's New?" dialog) — en, fr, de, it, es, pt, he |
| `src/inno setup/OpenStrand StudioX_XXX.iss` | New file. `MyAppVersion`, `MyAppDate`, `OutputBaseFilename` suffix, and the 7 `<lang>.WelcomeLabel2` lines |
| `src/build_installer_X_XXX.sh` | New file (macOS .pkg). `VERSION`, `APP_DATE`, and **49** `<ul>` bullet lists (7 welcome pages x 7 languages) |
| `src/build_dmg_X_XXX.sh` | New file (macOS .dmg). Same structure as the pkg script |
| `src/build_mac_X_XXX.sh` | New file. One-command macOS build; only version numbers change |
| `src/OpenStrandStudio_mac.spec` | `CFBundleShortVersionString` and `CFBundleVersion` |

## The easy way: the generator script

Everything above is generated by **`scripts/make_release_files.py`**. For each
release, edit only its CONFIG section:

1. `OLD_VERSION` / `NEW_VERSION` (e.g. `'1.109'` → `'1.110'`).
2. `OLD_DATE_SH`/`NEW_DATE_SH` (e.g. `'14_July_2026'`) and
   `OLD_DATE_ISS`/`NEW_DATE_ISS` (e.g. `'14_Jul_2026'` — abbreviated month).
3. `BULLETS`: the what's-new bullets as `(title, text)` pairs in **all 7
   languages** (en, fr, de, it, es, pt, he). Write plain unicode text —
   including Hebrew.
4. `MARKERS`: a short unique substring of the **previous** release's first
   bullet title per language (used to find the old bullet lists to replace).
   For Hebrew, use the `&#x....;` entity form as it appears in the old `.sh`.

Then:

```
python scripts/make_release_files.py
```

The script verifies that every language got the same number of replacements
(7 per macOS file) and that exactly 7 `whats_new_info` blocks were updated.

### Verify after generating

```
cd src
grep -c "1.109" build_installer_1_110.sh          # must be 0 (old version gone)
grep -c "<li>" build_installer_1_110.sh            # must be 49 x (number of bullets)
python -c "import ast; ast.parse(open('translations.py', encoding='utf-8').read())"
```

## Language / formatting rules (if you ever edit by hand)

- **7 languages everywhere**, in this order in each file: English, French,
  German, Italian, Spanish, Portuguese, Hebrew.
- **macOS welcome pages**: each `*.lproj/welcome.html` contains ALL 7 languages,
  with the target language FIRST (see the header comment inside the build
  scripts for the exact per-lproj order). That is why each `.sh` has 49 bullet
  lists — never update just one.
- **Hebrew**:
  - In the macOS `.sh` HTML pages Hebrew must be written as `&#x05D0;`-style
    HTML entities (ASCII stays literal), and the section sits inside
    `<div dir="rtl">`; the six LTR languages inside the Hebrew page use
    `<ul dir="ltr">`.
  - In the `.iss` and `translations.py`, Hebrew is plain UTF-8 text.
  - In `translations.py` the Hebrew block is wrapped in
    `<div dir="rtl" style="text-align: right;">`.
- **Copyright lines** in `translations.py` differ per language
  (`Version` / `Versione` / `Versión` / `– Versão` / `גרסה`) — keep them.
- **.iss**: bullets are `• Title: text` joined with `%n`; each
  `<lang>.WelcomeLabel2` is a single long line. Update `MyAppVersion`,
  `MyAppDate` (`DD_Mon_YYYY`), and the `_X_XXX` suffix in `OutputBaseFilename`.

## Building the installers

**Windows** (after `pyinstaller main.spec` produced `src/dist/OpenStrandStudio.exe`):
open `src/inno setup/OpenStrand StudioX_XXX.iss` in Inno Setup and compile.
Output goes to `src/dist/OpenStrandStudioSetup_<date>_X_XXX.exe`.

**macOS** (on a Mac, from `src/`):

```
chmod +x build_mac_1_110.sh
./build_mac_1_110.sh          # .app + .pkg
./build_mac_1_110.sh --dmg    # .app + .dmg instead
```

This runs PyInstaller with `OpenStrandStudio_mac.spec` (already version-bumped
by the generator) and then the pkg/dmg script. Output: `installer_output/`.
